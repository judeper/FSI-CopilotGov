    - Document RAC as a SAM capability for oversharing remediation:
      - RAC restricts access to SharePoint sites to only members of the site's associated security group
      - Unlike sharing permissions which grant additional access, RAC enforces a maximum access boundary
      - Particularly useful for sensitive sites that should only be accessible to a defined group, not anyone who happens to have a sharing link
    - Add configuration path: SharePoint Admin Center > Sites > Active sites > [site] > Settings > Restricted Access Control
    - Add tier recommendations: Baseline — deploy RAC on top 10 most sensitive sites; Recommended — RAC on all sites with Confidential or higher labels; Regulated — RAC on all sites containing NPI or MNPI with quarterly review
