    **Control 4.8 updates (P4-04):**
    - Document the **pay-as-you-go (PAYG) billing model** for Microsoft 365 Copilot:
      - Microsoft now offers a PAYG option for Microsoft 365 Copilot Chat at $0.01 per message (metered billing), in addition to the traditional per-seat license model.
      - This model is designed for organizations that want to provide Copilot Chat access to users who do not justify a full per-seat license — occasional users, seasonal workers, extended teams.
      - Billing is tracked through Azure Commerce and appears in the organization's Azure billing alongside other metered services.

    - Add a **per-seat vs PAYG cost comparison** table:

      | Factor | Per-Seat License | Pay-As-You-Go ($0.01/msg) |
      | Pricing model | Fixed monthly per-user cost | Variable cost based on actual usage |
      | Predictability | Highly predictable (fixed cost per user) | Variable — costs scale with usage |
      | Breakeven point | Cost-effective above ~N messages/user/month (Claude to calculate: $30/user/month / $0.01/msg = 3,000 messages) | Cost-effective below ~3,000 messages/user/month |
      | Features included | Full M365 Copilot (all apps, agents, Copilot Chat) | Copilot Chat only (not full in-app Copilot) |
      | Governance overhead | License assignment governance | Budget cap governance + usage monitoring |
      | FSI consideration | Predictable budget for SOX reporting | Requires budget controls to prevent cost overruns |
      | Audit billing | Included with E5 or E5 Compliance add-on | PAYG audit billing at $0.01/event for extended retention |

    - Document **PAYG governance controls**:
      - Budget caps: configure monthly spending limits per department or cost center to prevent uncontrolled cost growth
      - Usage alerts: set thresholds for PAYG usage notifications (e.g., alert at 80% of monthly budget)
      - Per-department tracking: allocate PAYG costs to departments using Azure cost management tags
      - Cost anomaly detection: monitor for unusual spikes in PAYG usage that could indicate misuse or unauthorized automation
      - Approval workflows: require management approval before enabling PAYG for user groups above certain size thresholds

    - Add FSI regulatory framing:
      - SOX 404: PAYG model requires budget authorization controls as part of IT general controls over financial reporting. Variable costs must be accurately tracked and allocated per SOX materiality thresholds.
      - OCC Heightened Standards: uncontrolled AI spending could indicate governance weakness. Budget caps and monitoring demonstrate responsible cost governance.
      - FFIEC: cost-benefit analysis for PAYG vs per-seat should be documented and periodically reviewed.

    - Update tier recommendations: Baseline: per-seat licensing only (maximum cost predictability); Recommended: per-seat for heavy users + PAYG for occasional users with budget caps; Regulated: per-seat for all regulated users + PAYG with strict budget caps, department-level cost allocation, and quarterly cost review.

    **All 4.8 playbooks:**
    - portal-walkthrough: Add navigation to Azure Commerce billing for PAYG monitoring. Add budget cap configuration steps in Azure Cost Management.
    - powershell-setup: Add Azure PowerShell cmdlets for PAYG cost monitoring, budget cap configuration, and usage reporting.
    - verification-testing: Add test cases for PAYG billing accuracy verification, budget cap enforcement verification, and cost allocation verification.
    - troubleshooting: Add common issues — PAYG costs not appearing in billing (delay), budget caps not enforcing (configuration scope), cost allocation tags not propagating.
