    Control 4.1: Copilot Admin Settings and Feature Management

    Control ID: 4.1 Pillar: Operations & Monitoring Regulatory Reference: SOX Section 404, FFIEC IT Examination Handbook, GLBA 501(b) Last Verified: 2026-02-17 Governance Levels: Baseline / Recommended / Regulated

    Objective

    Establish centralized governance over Microsoft 365 Copilot administrative settings and per-application feature toggles to maintain consistent operational controls, enforce segregation of duties for configuration changes, and support compliance with internal change management policies and regulatory expectations for technology governance.

    Why This Matters for FSI

    Financial institutions operate under rigorous internal control frameworks that require documented, auditable, and role-separated management of technology configurations. SOX Section 404 mandates that internal controls over financial reporting — including IT general controls — be documented, tested, and attested. The FFIEC IT Examination Handbook expects institutions to demonstrate that administrative access to critical systems is appropriately restricted and that configuration changes follow formal change management processes.

    The Microsoft 365 Copilot Control System (formerly the set of distributed Copilot admin controls across M365 Admin Center, Teams Admin Center, and individual app admin centers) provides a unified governance interface for all administrative capabilities across Microsoft 365 Copilot. Without centralized governance, different administrators may independently enable or disable Copilot features, leading to inconsistent security postures across business units, unmonitored feature rollouts, and gaps in audit evidence.

    For broker-dealers subject to FINRA oversight, uncontrolled Copilot enablement could expose regulated communications to AI-generated content without appropriate supervisory controls. For banks subject to OCC heightened standards, the lack of documented feature management processes could be cited during examinations as a control deficiency.

    SOX Section 404 IT general controls benefit directly from centralized monitoring dashboards that provide continuous evidence of ongoing governance oversight — the Copilot Control System overview dashboard serves precisely this function.

    Disclaimer

    This control is provided for informational purposes only and does not constitute legal, regulatory, or compliance advice. See full disclaimer.

    Control Description

    This control establishes a comprehensive governance framework for all Copilot administrative settings across the Microsoft 365 ecosystem. It covers:

    Copilot Control System

    The Copilot Control System is Microsoft's unified branding for all administrative governance capabilities across Microsoft 365 Copilot. It consolidates controls that previously required navigating across M365 Admin Center, Teams Admin Center, individual app admin centers, and Microsoft Purview into a coherent governance surface. The Copilot Control System encompasses the global Copilot toggle, per-application feature management, Baseline Security Mode, and the Copilot overview dashboard.

    Copilot Overview Dashboard

    Microsoft has introduced a dedicated Copilot dashboard at M365 Admin Center > Copilot > Overview that provides administrators with a consolidated governance view:

    • License assignment and utilization summary — current license deployment, active users, and utilization rates by department
    • Security posture overview — direct links to DSPM for AI, DLP policies covering Copilot surfaces, and sensitivity label coverage assessments
    • Usage analytics highlights — Copilot adoption trends and feature engagement metrics across the organization
    • Recommended actions — Microsoft-identified governance gaps with guided remediation steps

    This dashboard consolidates what previously required navigating multiple admin centers and separate reporting surfaces. For SOX 404 IT general controls, the overview dashboard provides a single point of evidence collection for demonstrating ongoing governance oversight of Copilot administrative posture.

    Portal Consolidation into MAC

    Microsoft is progressively consolidating Copilot governance controls into the Microsoft 365 Admin Center (MAC) under the Copilot section. Settings that previously required separate navigation to individual application admin centers are increasingly available directly from MAC > Copilot. The per-application feature toggles table below reflects settings that have migrated to the centralized Copilot section alongside settings that remain in their respective application admin centers:

    Application Primary Admin Center Key Copilot Settings
    Microsoft Teams MAC > Copilot / Teams Admin Center Meeting recap, chat compose assist, channel summaries
    Microsoft Outlook Exchange Admin Center Email drafting, email summarization, coaching tips
    Microsoft Word MAC > Copilot / M365 Apps Admin Document drafting, rewriting, summarization
    Microsoft Excel MAC > Copilot / M365 Apps Admin Formula generation, data analysis, PivotTable creation
    Microsoft PowerPoint MAC > Copilot / M365 Apps Admin Slide generation, design suggestions, summarization
    Microsoft OneNote MAC > Copilot / M365 Apps Admin Note summarization, to-do extraction
    Microsoft Loop MAC > Copilot Content generation, page summarization
    Microsoft Whiteboard MAC > Copilot Content generation in collaborative sessions
    SharePoint SharePoint Admin Center Site-level Copilot access, page summarization
    Microsoft Forms MAC > Copilot Survey generation assistance
    Microsoft Planner MAC > Copilot Task generation from meeting notes
    Microsoft 365 Copilot Chat MAC > Copilot Cross-app grounding, web grounding toggle

    Copilot for Admins

    Microsoft 365 Copilot is now available to assist administrators within the MAC, providing natural language configuration assistance, policy recommendation summaries, and troubleshooting guidance. Administrators can use Copilot for Admins to query the current state of Copilot policies, identify configuration gaps, and receive step-by-step guidance for complex configuration tasks.

    FSI Governance Note — Copilot-Assisted Admin Changes

    Copilot-assisted administrative changes still require the same change management approval process as any other configuration change. The Copilot for Admins AI assistant does not bypass CAB approval requirements, segregation of duties controls, or compliance officer review obligations. All recommendations generated by Copilot for Admins must be reviewed, approved, and documented through the standard change management workflow before implementation.

    Baseline Security Mode

    Microsoft offers a Baseline Security Mode for the Copilot Control System that applies a set of recommended security defaults as a starting configuration:

    • Sensitivity label requirements for Copilot interactions
    • DLP policies covering Copilot data access and output
    • Conditional Access policies for Copilot-licensed users
    • Recommended data residency and privacy settings

    Tier guidance for Baseline Security Mode:

    Tier Baseline Security Mode Approach
    Baseline Enable Baseline Security Mode as the starting configuration for all Copilot deployments
    Recommended Enable Baseline Security Mode and customize sensitivity label requirements and DLP policies per the firm's risk profile
    Regulated Enable Baseline Security Mode as a floor; fully customize all settings per regulatory obligations; document any deviations from baseline with business justification

    Global Copilot Toggle

    The Copilot Control System provides a tenant-wide toggle for Microsoft 365 Copilot. This master switch controls whether Copilot features are available to licensed users across the tenant. The primary Copilot settings page is located at M365 Admin Center > Copilot.

    Segregation of Duties

    Configuration changes to Copilot settings should require appropriate role separation:

    • M365 Global Admin or Copilot Admin role for tenant-wide Copilot settings
    • Teams Admin for Teams-specific Copilot features
    • Exchange Online Admin for Outlook Copilot features
    • SharePoint Admin for SharePoint Copilot features
    • Changes should be approved through a change advisory board (CAB) or equivalent process before implementation

    Feature Rollout Management

    Microsoft regularly introduces new Copilot features through targeted release rings. This control requires:

    • Monitoring Microsoft Message Center for upcoming Copilot feature changes
    • Evaluating new features against regulatory requirements before enablement
    • Using targeted release rings to test features with a pilot group before broad deployment
    • Documenting feature enablement decisions with regulatory impact assessments

    Copilot Surface Coverage

    Surface Coverage Notes
    Microsoft 365 Copilot Chat Full Global toggle + web grounding toggle
    Microsoft Teams Full Meeting recap, chat, channel features individually controllable
    Outlook Full Email composition and summarization toggles
    Word / Excel / PowerPoint Full Per-app toggles via MAC > Copilot / M365 Apps Admin Center
    SharePoint / OneDrive Full Site-level access controls available
    OneNote / Loop / Whiteboard Partial Settings managed at tenant level via MAC > Copilot
    Microsoft Forms / Planner Partial Limited individual toggle granularity

    Governance Levels

    Baseline

    • Enable Baseline Security Mode as the starting configuration for the Copilot Control System
    • Document all Copilot admin settings and their current state using the MAC > Copilot > Overview dashboard
    • Assign Copilot admin settings management to designated administrators with appropriate roles
    • Review global Copilot toggle state quarterly using the overview dashboard utilization summary
    • Maintain an inventory of which applications have Copilot enabled or disabled
    • Require documented approval before enabling Copilot in any new application

    Recommended

    • Implement segregation of duties — no single administrator can both approve and implement Copilot configuration changes
    • Configure Baseline Security Mode with firm-specific sensitivity label and DLP customizations
    • Configure targeted release ring with a pilot group of 20-50 users for new Copilot features
    • Establish monthly review of Copilot feature settings against organizational security policies using the overview dashboard
    • Create automated alerts for any changes to Copilot admin settings using Unified Audit Log monitoring
    • Maintain a configuration baseline document and compare against it during each review cycle
    • Document regulatory impact assessment for each Copilot feature toggle decision

    Regulated

    • Require dual approval (IT admin + compliance officer) for any Copilot feature enablement, including changes recommended by Copilot for Admins
    • Integrate Copilot configuration changes into the formal change management system (ServiceNow, Jira, etc.)
    • Capture configuration snapshots before and after every change for audit evidence; use overview dashboard security posture summary as pre/post evidence
    • Implement automated drift detection that alerts when Copilot settings deviate from the approved baseline
    • Include Copilot admin settings review in quarterly SOX ITGC testing procedures; use overview dashboard as primary evidence artifact
    • Maintain evidence of all configuration change approvals for a minimum of 7 years
    • Restrict Copilot admin role assignments using Privileged Identity Management (PIM) with time-bound activation

    Setup & Configuration

    Step 1: Access the Copilot Control System Dashboard

    Navigate to M365 Admin Center > Copilot > Overview to access the Copilot Control System dashboard. Review:

    1. License utilization summary — confirm licenses are assigned to approved user groups
    2. Security posture overview — identify any flagged gaps in sensitivity labels, DLP coverage, or Conditional Access
    3. Recommended actions — evaluate each recommendation against organizational risk profile before acting
    4. Usage analytics — review adoption trends for governance reporting

    Step 2: Enable Baseline Security Mode

    Navigate to M365 Admin Center > Copilot > Settings > Security:

    1. Enable Baseline Security Mode to apply Microsoft's recommended security defaults
    2. Review each default setting against organizational requirements
    3. Document any customizations applied beyond the baseline
    4. For Regulated tier: document any settings that deviate from the baseline with business justification

    Step 3: Inventory Current Copilot Settings

    Review all per-application Copilot settings from MAC > Copilot and document the current state. For each application-specific setting:

    Application: [Name]
Current State: Enabled / Disabled
Last Changed: [Date]
Changed By: [Admin UPN]
Approval Reference: [Ticket/CAB Number]

    Step 4: Define Role Assignments

    Map administrative roles to Copilot Control System responsibilities:

    Copilot Setting Area          Required Role              Backup Role
─────────────────────────────────────────────────────────────────────
Global Copilot Toggle         M365 Global Admin          Copilot Admin
Teams Copilot Features        Teams Admin                M365 Global Admin
Outlook Copilot Features      Exchange Online Admin      M365 Global Admin
SharePoint Copilot Features   SharePoint Admin           M365 Global Admin
M365 Apps (Word/Excel/PPT)    M365 Apps Admin            M365 Global Admin
Baseline Security Mode        Copilot Admin              M365 Global Admin

    Step 5: Configure Change Monitoring

    Enable audit logging for Copilot administrative actions:

    1. Navigate to Microsoft Purview > Audit > Audit Search
    2. Search for activities containing "Copilot" in the workload filter
    3. Key operations to monitor: Set-CopilotPolicy, Update-CopilotSettings, admin center changes
    4. Create a recurring saved search and configure alert policies

    Step 6: Establish Targeted Release

    1. Navigate to M365 Admin Center > Settings > Org Settings > Release Preferences
    2. Select Targeted release for selected users
    3. Add pilot group members (compliance-aware users from IT, legal, and business units)
    4. Document the pilot group membership and review criteria

    Financial Sector Considerations

    Broker-Dealers (FINRA-regulated): Before enabling Copilot in Outlook or Teams for registered representatives, verify that supervisory review systems (Control 3.x) are configured to capture and review AI-assisted communications. Enabling Copilot for communication tools without corresponding supervisory controls may create supervisory gaps that examiners could identify during routine examinations.

    Banking Institutions (OCC/FDIC-regulated): Document Copilot feature enablement decisions as part of the institution's technology risk assessment. OCC heightened standards expect boards and senior management to maintain awareness of new technology deployments that could affect operational risk profiles.

    Insurance Carriers (State-regulated): Consider state-specific requirements for AI usage disclosure. Some states require notification when AI tools are used in claims processing or underwriting. If Copilot is enabled in applications used for these functions, document the disclosure approach.

    Investment Advisers (SEC-regulated): Copilot features in Excel and Word that generate financial analysis or client-facing content should be enabled only after establishing appropriate review procedures consistent with the adviser's fiduciary obligations.

    SOX-Reporting Entities: All Copilot Control System configuration changes that could affect financial reporting workflows (Word, Excel, Outlook for financial communications) should be included in the SOX ITGC scope and tested during annual attestation procedures. The MAC > Copilot > Overview dashboard provides centralized evidence of governance oversight for SOX 404 attestation purposes.

    Verification Criteria

    # Verification Step Expected Result
    1 Navigate to MAC > Copilot > Overview dashboard Dashboard accessible; license utilization and security posture sections visible
    2 Review Copilot Control System settings All toggles match documented configuration baseline
    3 Verify Baseline Security Mode status Baseline Security Mode enabled; customizations documented
    4 Verify role assignments in Entra ID for Copilot admin roles Only designated administrators hold Copilot-related admin roles
    5 Search Unified Audit Log for Copilot setting changes in the past 90 days All changes have corresponding approved change requests
    6 Review targeted release ring configuration Pilot group is defined and includes compliance-aware users
    7 Confirm configuration register is current Register updated within the last 30 days
    8 Verify PIM configuration for admin roles (Regulated) Admin roles require time-bound activation with justification
    9 Test segregation of duties No single user can both approve and implement Copilot changes
    10 Confirm alert policies for setting changes Alerts trigger on Copilot configuration modifications
