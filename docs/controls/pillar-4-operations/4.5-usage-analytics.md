# Control 4.5: Copilot Usage Analytics and Adoption Reporting

**Control ID:** 4.5
**Pillar:** Operations & Monitoring
**Regulatory Reference:** SOX Section 404 (Control Effectiveness Monitoring), FFIEC IT Examination Handbook, GLBA 501(b)
**Last Verified:** 2026-02-17
**Governance Levels:** Baseline / Recommended / Regulated

---

## Objective

Implement structured monitoring and reporting of Microsoft 365 Copilot usage analytics to measure control effectiveness, track adoption patterns across business units, identify anomalous usage that may indicate compliance gaps, and provide evidence of ongoing governance to regulators and auditors.

## Why This Matters for FSI

Regulatory frameworks applicable to financial services institutions require not just the implementation of controls, but ongoing evidence that those controls are operating effectively. SOX Section 404 mandates that management assess and report on the effectiveness of internal controls over financial reporting, which extends to IT general controls governing AI tools used in financial workflows.

Usage analytics serve multiple governance purposes in regulated environments:

- **Control Effectiveness Evidence:** If Copilot governance controls are in place but usage data shows that users are bypassing or circumventing them, the controls are not operating effectively. Usage analytics help detect these gaps.
- **Supervisory Adequacy:** FINRA Rule 3110 requires supervisory systems that are reasonably designed for the firm's business model. Usage analytics inform whether supervisory review coverage is adequate relative to actual Copilot usage volumes.
- **Risk Proportionality:** The FFIEC expects institutions to allocate risk management resources proportional to the risk. Usage data helps determine where Copilot is most heavily used and whether governance investments are appropriately aligned.
- **Examination Readiness:** Regulators increasingly request data on AI tool usage during examinations. Having structured usage reports readily available demonstrates proactive governance.

Without usage analytics, financial institutions cannot demonstrate that their Copilot governance framework is more than a paper exercise. Examiners look for evidence of monitoring, not just the existence of policies.

## Control Description

**M365 Admin Center Copilot Usage Reports**

The M365 Admin Center provides built-in Copilot usage reports:

| Report | Description | Key Metrics |
|--------|-------------|-------------|
| Copilot Usage Summary | Tenant-wide Copilot adoption overview | Active users, activation rate, feature usage by app |
| Copilot in Teams | Teams-specific Copilot usage | Meeting summaries generated, chat assists, channel summaries |
| Copilot in Outlook | Outlook-specific Copilot usage | Emails drafted, emails summarized, coaching suggestions |
| Copilot in Word | Word-specific Copilot usage | Documents drafted, rewritten, summarized |
| Copilot in Excel | Excel-specific Copilot usage | Formulas generated, data analyses performed |
| Copilot in PowerPoint | PowerPoint-specific Copilot usage | Slides generated, presentations summarized |
| Copilot in BizChat | Business Chat usage | Cross-app queries, web-grounded queries |
| Copilot Readiness | License assignment and activation | Assigned licenses, activated users, unactivated licenses |

**Microsoft 365 Usage Analytics**

The broader M365 usage analytics platform provides additional context:

- **Adoption Score:** Organization-wide productivity and technology adoption metrics
- **Product Usage Trends:** Historical usage patterns across all M365 workloads
- **User Activity Reports:** Individual and group-level activity data (subject to privacy controls)

**Department-Level Usage Tracking**

For governance purposes, usage should be segmented by organizational unit:

| Segmentation | Purpose | Implementation |
|-------------|---------|----------------|
| By Department | Identify which business units use Copilot most heavily | Entra ID group-based reporting |
| By Role Type | Distinguish regulated vs. non-regulated user adoption | Custom groups or Entra attributes |
| By Geography | Track usage across jurisdictions for regulatory alignment | Entra ID location attributes |
| By License Type | Monitor utilization of assigned licenses | License assignment reports |
| By Risk Tier | Correlate usage with risk classifications | Custom mapping to risk tiers |

**Active User Tracking**

Active user metrics provide the foundation for usage governance:

- **Monthly Active Users (MAU):** Users who used Copilot at least once in 30 days
- **Weekly Active Users (WAU):** Users who used Copilot at least once in 7 days
- **Daily Active Users (DAU):** Users who used Copilot on a given day
- **Feature-Specific Active Users:** Users who used a specific Copilot feature
- **Inactive Licensed Users:** Users with Copilot licenses who have not used the service (license optimization candidates)

## Governance Levels

### Baseline

- [ ] Enable M365 Admin Center Copilot usage reports
- [ ] Establish monthly review of Copilot usage summary reports by IT governance team
- [ ] Track license assignment vs. active usage rates to identify optimization opportunities
- [ ] Document which administrators have access to usage reports and analytics
- [ ] Maintain a quarterly usage trend summary for governance committee reporting
- [ ] Set baseline usage benchmarks against which future periods are compared

### Recommended

- [ ] Implement department-level usage segmentation using Entra ID group-based reporting
- [ ] Create automated usage report distribution to department heads and compliance officers
- [ ] Establish usage anomaly thresholds and alerting (e.g., sudden usage spikes or drops)
- [ ] Correlate Copilot usage data with supervisory review volumes to assess coverage adequacy
- [ ] Track adoption curves for new Copilot features to assess change management effectiveness
- [ ] Generate monthly governance dashboards combining usage, compliance, and risk metrics
- [ ] Compare usage patterns across similarly regulated peer departments to identify outliers

### Regulated

- [ ] Include Copilot usage analytics in SOX ITGC testing evidence packages
- [ ] Present quarterly Copilot usage reports to the board risk committee or technology committee
- [ ] Implement automated correlation between usage patterns and compliance events
- [ ] Maintain 7-year archive of usage analytics for examination readiness
- [ ] Conduct annual assessment of whether supervisory resources are proportional to Copilot usage volumes
- [ ] Integrate usage analytics with the firm's GRC (Governance, Risk, Compliance) platform
- [ ] Document usage analytics methodology and data sources for auditor review

## Setup & Configuration

### Step 1: Enable Usage Reporting

1. Navigate to **M365 Admin Center > Reports > Usage**
2. Verify that Copilot reports are visible (requires Copilot licenses assigned in the tenant)
3. Configure report visibility settings:
   - Determine whether individual user names are shown or anonymized
   - For privacy, consider anonymizing individual user data in admin reports
   - For compliance, designated compliance officers may need named user data

### Step 2: Configure Report Access

Define who can access Copilot usage reports:

```
Role                        Access Level           Justification
─────────────────────────────────────────────────────────────────
M365 Global Admin           Full                   Tenant administration
Reports Reader              Summary reports        Governance reporting
Usage Summary Reports       Department heads       Adoption oversight
Reader
Compliance Admin            Full (named users)     Supervisory adequacy
                                                   assessment
```

Assign the **Reports Reader** role in **Entra ID > Roles and Administrators** to designated governance personnel.

### Step 3: Establish Reporting Cadence

| Report | Frequency | Audience | Owner |
|--------|-----------|----------|-------|
| Usage Summary Dashboard | Monthly | IT Governance, CISO | IT Operations |
| Department Adoption Report | Monthly | Department Heads | Copilot Program Manager |
| License Optimization Report | Quarterly | Finance, IT Procurement | License Manager |
| Supervisory Coverage Analysis | Quarterly | Compliance | Chief Compliance Officer |
| Board-Level AI Usage Report | Quarterly | Board Risk Committee | CTO/CIO |
| Annual Usage Trend Analysis | Annually | Executive Leadership | Copilot Program Manager |

### Step 4: Configure Anomaly Detection

Establish thresholds for usage anomalies that warrant investigation:

| Anomaly | Threshold | Action |
|---------|-----------|--------|
| Usage spike | >200% increase in department usage over 7 days | Investigate for unauthorized enablement |
| Usage drop | >50% decrease in active users over 30 days | Verify service availability and user satisfaction |
| After-hours usage | Significant Copilot usage outside business hours | Review for potential data exfiltration risk |
| Concentrated usage | Single user generating >10x average Copilot interactions | Evaluate for automated/scripted access |
| New feature adoption | Rapid adoption of newly released feature | Verify feature has been through governance review |

### Step 5: Build Governance Dashboard

Create a consolidated governance dashboard using available tools:

1. **Power BI + M365 Usage Analytics:** Connect Power BI to M365 usage data for custom visualizations
2. Include the following panels:
   - Active user trends (DAU/WAU/MAU)
   - Usage by application and department
   - License utilization rate
   - Compliance event correlation
   - Adoption vs. governance milestone timeline

## Financial Sector Considerations

**SOX Attestation Evidence:** For public financial institutions subject to SOX, Copilot usage analytics provide evidence that AI governance controls are operating effectively. Include usage reports in ITGC evidence packages alongside control testing results. Auditors expect to see monitoring data, not just policy documents.

**Examination Readiness:** Regulators (FINRA, OCC, FDIC, state regulators) are increasingly interested in how financial institutions deploy and monitor AI tools. Having structured usage analytics readily available demonstrates mature governance. Prepare a standardized examination response package that includes usage trends, governance decisions, and control effectiveness metrics.

**Supervisory Resource Planning:** For broker-dealers, correlating Copilot usage in communication channels (Teams, Outlook) with supervisory review volumes helps determine whether review resources are adequate. If Copilot is generating thousands of AI-assisted communications monthly but supervisory review covers only a fraction, the firm may need to adjust sampling rates or review technology.

**License Cost Justification:** Financial institutions face scrutiny on technology spending. Usage analytics that show low adoption or inactive licenses provide data-driven justification for license reallocation, supporting the institution's fiduciary obligations to shareholders (for public companies) or members (for credit unions).

**Peer Benchmarking:** While external benchmarking data may be limited, internal peer comparison across departments with similar regulatory profiles helps identify governance outliers. If the trading desk shows dramatically different usage patterns from wealth management, that may warrant investigation.
