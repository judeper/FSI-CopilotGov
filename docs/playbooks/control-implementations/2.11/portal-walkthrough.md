# Control 2.11: Copilot Pages Security and Sharing Controls — Portal Walkthrough

Step-by-step portal configuration for securing Copilot Pages, the collaborative content surface generated by Copilot interactions.

## Prerequisites

- SharePoint Administrator role
- Microsoft 365 Copilot licenses deployed
- Understanding of Copilot Pages functionality and data flow
- Governance committee decision on Pages sharing policy

## Steps

### Step 1: Review Copilot Pages Storage Location

**Portal:** SharePoint Admin Center
**Path:** SharePoint Admin > Sites > Active Sites > filter for "Copilot Pages"

Copilot Pages are stored in a dedicated SharePoint Embedded location. Review the storage configuration and understand how Pages content is organized, shared, and retained.

### Step 2: Configure Pages Sharing Defaults

**Portal:** Microsoft 365 Admin Center
**Path:** Admin Center > Settings > Microsoft 365 Copilot > Pages

Configure the default sharing behavior for Copilot Pages:
- **Recommended for FSI:** Limit sharing to "People with existing access" or "Specific people"
- Disable "Anyone with the link" sharing for Pages
- Set default sharing scope to minimize unintended data exposure
- Configure whether Pages can be shared externally (recommend: disabled for FSI)

### Step 3: Apply Sensitivity Labels to Pages

**Portal:** Microsoft Purview
**Path:** Purview > Information Protection > Label Policies

Verify that sensitivity labels apply to Copilot Pages content. Pages generated from labeled source content should inherit the appropriate label. Configure:
- Default label for new Pages
- Label inheritance from source Copilot interactions
- Mandatory labeling for Pages if supported

### Step 4: Configure Retention Policies for Pages

**Portal:** Microsoft Purview
**Path:** Purview > Data Lifecycle Management > Retention Policies

Create or extend retention policies to cover Copilot Pages:
- Define retention period aligned with records management requirements
- Configure automatic deletion after retention period for non-record Pages
- Ensure Pages containing regulated content are subject to appropriate holds

### Step 5: Monitor Pages Usage and Sharing

Set up monitoring to track how Copilot Pages are created and shared:
- Review Pages creation volume and sharing patterns
- Monitor for Pages containing sensitive content shared broadly
- Track external sharing attempts (if not blocked at the policy level)

## FSI Recommendations

| Tier | Recommendation |
|------|---------------|
| **Baseline** | Limit Pages sharing to "Specific people"; disable external sharing |
| **Recommended** | Sensitivity label inheritance on Pages; retention policies applied; sharing monitoring |
| **Regulated** | Mandatory labeling; no external sharing; DLP policies on Pages content; comprehensive audit logging |
